# Web システム要件定義

## 概要

自分が選挙で投票した政党や候補者を記録として残し、後から参照できる Web システムを開発する。

## システム要件

### 機能要件

- Google アカウントによる OAuth 認証
- 認証後、自分が選挙で投票した政党や候補者を登録できる
  - 登録項目：
    - 選挙日
    - 選挙の種類（衆院選挙、参院選挙、都道府県長選挙、市長選挙など）
    - 選挙名
    - 政党名
    - 候補者名
    - 投票写真（投票記録としてアップロード可能）
- 過去に登録した政党・候補者・投票記録を参照できる（自分のデータのみ閲覧可）
- 候補者の当時のマニフェストや、投票以降の実績を参照できる
  - マニフェスト・実績情報は Web サイトをクローリングして収集（API があれば API 利用も検討）
- 投票記録（候補者登録時）を SNS（X、Facebook）でシェアできる

### 非機能要件

- データ保存は AWS 上のクラウド DB を利用
- 認証後は自分のデータのみ閲覧可能（プライバシー保護）
- モバイル端末からの利用に対応（レスポンシブデザイン）
- 想定ユーザー数：数十人〜100 人以内
- 管理者機能（データ管理、不正利用監視等）は必要だが優先度は低い
- アクセシビリティ要件は特に設けない

### データモデル（案）

- ユーザー
  - Google アカウント ID
  - 氏名（任意）
  - 出身地域
- 選挙
  - 選挙 ID
  - 選挙名
  - 選挙日
  - 選挙種類
- 政党
  - 政党 ID
  - 政党名
- 候補者
  - 候補者 ID
  - 氏名
  - 所属政党
  - マニフェスト URL
  - 実績情報
- 投票記録
  - ユーザー ID
  - 選挙 ID
  - 候補者 ID
  - 投票日
  - 投票写真 URL

### セキュリティ・プライバシー

- Google OAuth による認証必須
- 各ユーザーは自分の投票記録のみ閲覧可能
- データは AWS クラウド DB 上で管理し、適切なアクセス制御を実施

### その他

- システムの拡張性を考慮し、将来的に他の SNS や外部 API 連携も可能な設計とする
- バックアップや障害対応については AWS の標準機能を活用
